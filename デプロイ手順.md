# GitHub & Render デプロイ手順

新しいプロジェクトをゼロから公開するときの手順書。

---

## 事前準備（初回のみ）

### Git のユーザー設定

```bash
git config --global user.name "GitHubのユーザー名"
git config --global user.email "GitHubに登録しているメールアドレス"
```

設定済みか確認するには：

```bash
git config --global user.name
git config --global user.email
```

---

## Step 1 ─ .gitignore を作る

`node_modules/` など、GitHubにあげてはいけないファイルを除外する。

```
# .gitignore
node_modules/
.env
```

> **注意**  
> `.gitignore` を作る前に `git add` してしまうと `node_modules/` がアップロードされてしまう。  
> 必ず最初に作ること。

もし先にアップロードしてしまった場合は以下で削除できる：

```bash
git rm -r --cached node_modules
git add .gitignore
git commit -m "remove node_modules, add .gitignore"
git push
```

---

## Step 2 ─ GitHub にリポジトリを作る

1. [github.com](https://github.com) を開く
2. 左上の「**New**」ボタンをクリック
3. 以下の設定でリポジトリを作成する

| 項目 | 設定値 |
|------|--------|
| Repository name | プロジェクト名（例: gps-tracker） |
| Visibility | Public または Private |
| Add a README | **チェックしない** |
| Add .gitignore | **チェックしない**（自分で作るため） |

4. 「**Create repository**」をクリック

---

## Step 3 ─ ローカルのコードを GitHub に push する

ターミナルでプロジェクトフォルダに移動してから、以下を順番に実行する。

```bash
# Git リポジトリを初期化する
git init

# 全ファイルをステージングに追加する
git add .

# コミット（変更内容を記録）する
git commit -m "first commit"

# ブランチ名を main に変更する
git branch -M main

# GitHub のリポジトリと紐づける（URLは自分のものに変える）
git remote add origin https://github.com/ユーザー名/リポジトリ名.git

# GitHub に push する
git push -u origin main
```

ブラウザで認証を求められたら許可する。

---

## Step 4 ─ Render にデプロイする

1. [render.com](https://render.com) を開き、「**Sign in with GitHub**」でログイン
2. 右上の「**+ New**」→「**Web Service**」をクリック
3. リポジトリ一覧から対象のリポジトリを選択
4. 以下の設定を確認・入力する

| 項目 | 設定値 |
|------|--------|
| Language | Node |
| Branch | main |
| Build Command | `npm install` |
| Start Command | `node server.js` |
| Instance Type | **Free**（$0/month） |

5. 「**Deploy Web Service**」をクリック

デプロイ完了まで1〜2分かかる。完了すると以下のようなURLが発行される：

```
https://プロジェクト名-xxxx.onrender.com
```

---

## Step 5 ─ 2回目以降の更新手順

コードを修正したあとは以下の3コマンドだけでOK。  
GitHub に push すると Render が自動で再デプロイしてくれる。

```bash
git add .
git commit -m "変更内容のメモ"
git push
```

## よくあるトラブル

### node_modules がアップロードされてしまった

```bash
git rm -r --cached node_modules
echo node_modules/ > .gitignore
git add .gitignore
git commit -m "remove node_modules, add .gitignore"
git push
```

### Render でデプロイに失敗する

Render の「**Logs**」タブでエラー内容を確認する。よくある原因：

- `Start Command` が間違っている（`node server.js` になっているか確認）
- `package.json` の `scripts.start` が設定されていない
- `node_modules/` がなく、`Build Command` が `npm install` になっていない

### Render の無料プランでページが開くのが遅い

15分間アクセスがないとサーバーがスリープする。  
再アクセス時に起動まで50秒ほどかかる場合がある。有料プランにすれば解消する。
