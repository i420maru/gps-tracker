# GPS トラッカー 開発ロードマップ

スマホの位置情報をリアルタイムでPCの地図上に表示するWebアプリ。

---

## システム概要

```
スマホ（ブラウザ）
  └─ GPS取得（Geolocation API）
  └─ WebSocket でサーバーへ送信
         ↓
  サーバー（Node.js / Render）
  └─ 受け取った座標を全クライアントに転送（ブロードキャスト）
         ↓
  PC（ブラウザ）
  └─ WebSocket で座標を受信
  └─ Leaflet.js で地図上にピンを表示
```

---

## ファイル構成

```
gps-tracker/
├── server.js          ← Node.js サーバー（WebSocket も兼ねる）
├── public/
│   ├── index.html     ← PC で開く地図ページ
│   └── phone.html     ← スマホで開く GPS 送信ページ
└── package.json       ← 依存ライブラリの定義
```

## 各ステップの詳細

### Step 1 ─ package.json を作る

使うライブラリは2つだけ。

- **express** ─ Webサーバー。`public/` フォルダの HTML ファイルをブラウザに届ける
- **ws** ─ WebSocket サーバー。スマホ↔サーバー↔PC のリアルタイム通信を担う

---

### Step 2 ─ サーバーの土台を作る

Express を使って `public/` フォルダの静的ファイルを配信する。  
この時点で `http://localhost:3000/` にアクセスすれば HTML が表示される。

---

### Step 3 ─ WebSocket を追加する

Express の HTTP サーバーに WebSocket サーバーを「相乗り」させる。  
スマホから届いた JSON（緯度・経度）を、接続中の全クライアントに転送（ブロードキャスト）する。

```
受信した JSON の形:
{
  "type": "location",
  "deviceId": "device_abc123",
  "deviceName": "田中のスマホ",
  "lat": 33.8416,
  "lng": 132.7657,
  "accuracy": 15,
  "timestamp": "2025-04-01T10:00:00.000Z"
}
```

---

### Step 4 ─ スマホ送信ページを作る（phone.html）

1. `navigator.geolocation.watchPosition()` で GPS を監視
2. 5秒ごとに最新座標を WebSocket でサーバーへ送信
3. 接続状態・送信回数・現在座標を画面に表示

> **HTTPS が必要**  
> ブラウザのセキュリティ仕様により、Geolocation API は HTTPS 環境でしか動かない（localhost は例外）。  
> Render にデプロイすれば自動的に HTTPS になるため問題ない。

---

### Step 5 ─ PC地図ページを作る（index.html）

1. Leaflet.js（CDN）で地図を表示
2. WebSocket でサーバーに接続、座標を受信
3. デバイスごとにマーカーを管理し、新しい座標が来たら位置を更新

---

### Step 6 ─ ローカルで動作確認

```bash
node server.js
```

- PC ブラウザ → `http://localhost:3000/`（地図ページ）
- スマホブラウザ → `http://＜PCのローカルIP＞:3000/phone.html`
  - スマホと PC を **同じ Wi-Fi** につないでおく必要がある
  - PCのローカルIPは `ipconfig`（Windows）で確認

---

### Step 7 ─ Render にデプロイ

1. GitHub にリポジトリを作って push
2. [render.com](https://render.com) で "New Web Service" を作成
3. リポジトリを連携、Start Command を `node server.js` に設定
4. デプロイ完了 → `https://＜サービス名＞.onrender.com` で公開

> **Render 無料プランの注意点**  
> 15分間アクセスがないとサーバーがスリープする。  
> 再アクセス時に起動まで30秒ほどかかる場合がある。

---

### Step 8 ─ スマホから本番接続

- `https://＜サービス名＞.onrender.com/phone.html` をスマホで開く
- 位置情報の許可を求めるダイアログが出るので「許可」
- PC で `https://＜サービス名＞.onrender.com/` を開いて地図を確認
